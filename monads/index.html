Monads

A monad can be thought of as a simple interface. Rather than giving a lengthy and winding explanation, of which you can find _so so many_, I want to just show you a Monad in a language you might recognize. I believe just seeing the code can give you the context to understand lengthier explanations like this nice one from Wikipedia:

  • Monad (functional programming)

Disclaimer

This is a "Monad" as implemented in Haskell and similar FP languages. That's not the same as a "monad" from category theory. Learning the difference is left as exercise for the reader.

Languages

  • Java
  • Python
  • TypeScript

I'll update occasionally with other languages as time permits.

The examples favor languages with some kind of typing. I don't have anything against languages without explicit types, I've even written one of my own, but it's much harder to directly communicate the Monad pattern without types.

Ok let's do this!

Java


import java.util.function.Function;

public interface Monad {
    Monad unit(A a);
    Monad bind(Function> f);
}

Python


from abc import ABC
from typing import Callable, Generic, TypeVar

A = TypeVar('A')
B = TypeVar('B')

class Monad(ABC, Generic[A]):
    @abstractmethod
    def unit(A) -> Monad[A]:
        raise NotImplementedError("unit")

    @abstractmethod
    def bind(self, f: Callable[A, Monad[B]]) -> Monad[B]:
        raise NotImplementedError("bind")

TypeScript


export interface Monad {
    unit(a: A) => Monad;
    bind(f: (a: A) => Monad) => Monad;
};
